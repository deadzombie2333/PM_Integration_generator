# 退款通知 (Refund Notification)

## API Information

- **Method**: POST (Callback from PayerMax)
- **Endpoint**: `https://pay-gate-uat.payermax.com/RefundResultNotifyUrl`
- **Content-Type**: `application/json`

## Description

退款完成后，PayerMax会把退款结果通过数据流的形式发送给商户，商户需要接收处理，并按文档规范返回应答。

**重要提示**：
- 该地址无需商户主动请求
- 以下request.body为PayerMax对退款状态的回调内容
- 商户收到回调后需要响应：`{"msg": "Success","code": "SUCCESS"}`

## Request Body (Callback from PayerMax)

### Root Level Parameters

| Parameter | Type | Required | Max Length | Description |
|-----------|------|----------|------------|-------------|
| code | string | ✓ | 32 | 返回码，'APPLY_SUCCESS'代表成功 |
| msg | string | ✓ | 256 | 响应描述，'Success.' |
| keyVersion | string | ✓ | 8 | 密钥版本。当前值为：1 |
| appId | string | ✗ | 64 | 商户app id |
| merchantNo | string | ✓ | 15 | 商户Id |
| notifyTime | string | ✓ | 32 | 请求时间，符合rfc3339规范，格式：yyyy-MM-dd'T'HH:mm:ss.SSSXXX |
| notifyType | string | ✓ | 16 | 通知类型 REFUND |
| data | object | ✓ | - | 数据体 |

### Data Object Parameters

| Parameter | Type | Required | Max Length | Description |
|-----------|------|----------|------------|-------------|
| outRefundNo | string | ✓ | 64 | 商户退款单号 |
| refundTradeNo | string | ✓ | 64 | PayerMax退款流水号 |
| outTradeNo | string | ✓ | 64 | 原商户订单号 |
| refundAmount | number | ✓ | - | 退款金额，金额的单位为元。 |
| refundCurrency | string | ✓ | 3 | 退款币种 |
| status | string | ✓ | 32 | 退款单状态: REFUND_SUCCESS，REFUND_FAILED |
| refundFinishTime | string | ✓ | - | 退款完成时间，utc+0时区。 |

### Notify Type Value

| Value | Description |
|-------|-------------|
| REFUND | 退款通知 |

### Refund Status Values

| Value | Description |
|-------|-------------|
| REFUND_SUCCESS | 退款成功 |
| REFUND_FAILED | 退款失败 |

## Response (200) - Merchant Response to PayerMax

### Response Body

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| code | string | ✓ | 返回码，'SUCCESS'代表成功 |
| msg | string | ✓ | 返回描述，'Success.' |

### Example Response

```json
{
  "code": "SUCCESS",
  "msg": "Success"
}
```

## Notes

- ✓ = Required field
- ✗ = Optional field
- 这是一个回调接口，由PayerMax主动调用商户的refundNotifyUrl
- 商户必须验证签名以确保回调来自PayerMax
- 商户必须返回正确的响应格式，否则PayerMax会重试回调
- notifyType固定为REFUND
- refundFinishTime使用UTC+0时区
