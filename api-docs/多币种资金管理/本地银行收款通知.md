### Body (application/json)

| 参数名 | 类型 | 必填 | 说明 | 限制 |
|--------|------|------|------|------|
| code | string | 是 | 返回码,'APPLY_SUCCESS'代表有效通知 | <= 32 characters |
| msg | string | 是 | 有效通知描述:Success | - |
| version | string | 是 | 接口版本,当前值为:1.0 | - |
| keyVersion | string | 是 | 密钥版本,当前值为:1 | <= 8 characters |
| appId | string | 是 | 商户应用Id,PayerMax分配给商户应用的唯一标识 | <= 32 characters |
| merchantNo | string | 是 | 商户号,商户与PayerMax业务签约时生成的唯一标识 | <= 15 characters |
| notifyTime | string | 是 | 通知时间,符合rfc3339规范,格式:yyyy-MM-dd'T'HH:mm:ss.SSSXXX | <= 32 characters |
| notifyType | string | 是 | 通知类型,RECEIVE | <= 16 characters |
| data | object | 是 | 数据体 | - |

#### data 对象

| 参数名 | 类型 | 必填 | 说明 | 限制 |
|--------|------|------|------|------|
| requestNo | string | 是 | PayerMax交易流水号 | <= 64 characters |
| channelTradeNo | string | 是 | 银行单号/外部订单号 | <= 64 characters |
| status | string | 是 | TO_BE_PROVED 需要补交材料; SETTLED 交易成功已入账; REJECTED 交易失败已退回 | <= 16 characters |
| transactionUtcTime | string | 是 | 交易创建时间,符合rfc3339规范,格式:yyyy-MM-dd'T'HH:mm:ss.SSSXXX | <= 32 characters |
| completeTime | string | 是 | 交易完成时间,符合rfc3339规范,格式:yyyy-MM-dd'T'HH:mm:ss.SSSXXX | <= 32 characters |
| paymentDetail | object | 是 | 支付要素 | - |
| errorCode | string | 是 | 订单失败错误码,详见[错误码列表] | <= 32 characters |
| errorMsg | string | 是 | 订单失败错误码描述 | <= 256 characters |

## Response

### 200 成功响应

#### Body (application/json)

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| msg | string | 是 | 响应消息 |
| code | string | 是 | 响应码 |
