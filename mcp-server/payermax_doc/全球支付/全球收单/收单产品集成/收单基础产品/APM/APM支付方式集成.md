# APM支付方式集成

APM 的全称是 **Alternative Payment Methods（替代支付方式）**，泛指除现金、银行卡、ApplePay、GooglePay之外的其他支付方式。

## 什么是 APM

APM包括但不限于：
- **移动钱包**: 如 DANA移动钱包
- **银行转账**: 直接银行转账支付
- **先买后付**: 如 KLARNA

## 1. 支持的支付方式

当前PayerMax支持的APM支付方式参看**支付方式列表**。

不同APM支付方式可支持的集成模式、退款能力存在较大的差异，具体可通过支付方式列表中的相关能力及集成模式进行查看。

## 2. 集成说明

相比卡支付，APM支付方式所需的支付信息和流程通常更少，如：
- 无3DS认证
- 无PCI资质要求

### 集成模式

APM方式在不同集成模式下的详细集成流程，可参看集成模式对接章节：

- **收银台模式集成**
- **纯API支付集成**
- **链接支付集成**

## 优势

APM支付方式相比传统卡支付具有以下优势：

1. **更简单的集成流程** - 无需复杂的安全认证
2. **更低的合规要求** - 无需PCI DSS认证
3. **更快的支付体验** - 减少认证步骤
4. **本地化支付** - 支持各地区流行的支付方式
