# 卡支付简介

银行卡支付，简称卡支付，是基于银行卡片的支付方式，广泛应用于现代社会的线上或线下支付。

## 1. PayerMax支持的卡组

PayerMax支持国际卡组Visa、MasterCard、JCB；以及覆盖沙特阿拉伯、韩国、巴西等不同国家的本地卡组。

不同卡品牌支持的国家、下单币种、最小交易金额等信息请查看支付方式列表。

### 支持的卡组织
- **国际卡组**: Visa、MasterCard、JCB
- **本地卡组**: 覆盖沙特阿拉伯、韩国、巴西等不同国家

## 2. 卡支付的安全保障

为保障卡支付的安全性，避免卡信息泄露导致的欺诈，卡组织对卡支付的流程有严格的安全要求，包括：

### PCI DSS
规范卡信息的收集、处理、存储或传输的数据环境。更多信息，请查看PCI DSS合规指南。

### 3DS认证
规范在线支付下的无卡交易，提供额外的验证流程，区分支付欺诈的责任。更多信息查看3DS认证。

PayerMax支付系统已经过PCI DSS认证，无论商户是否有PCI DSS资质，均可实现卡支付。

不同集成模式的卡支付，对商户的PCI DSS资质要求，请查看集成模式概览。

## 3. Tokenization卡支付

使用PayerMax支付系统，您可以在购物者同意的情况下，可以将每个购物者的每张卡的卡要素信息（如：卡号、CVV、有效期等）以令牌（Token）形式保存在PayerMax系统。

这样，当用户后续使用该卡进行支付时，可以无须传递卡要素信息，仅传递Token即可完成支付。

### Tokenization的优势
- 简化用户支付流程
- 提高支付成功率
- 适用于无PCI资质但希望集成纯API支付的商户

关于卡支付下使用Token完成支付，更多信息请查看Tokenization。

## 4. 基于推荐卡进行支付

当用户成功完成一笔卡支付后，PayerMax会保存该笔支付中用户使用的卡信息。当用户下次支付时，PayerMax收银页会展示该用户历史支付成功的所有卡（以卡号掩码形式展示）。

对于已存卡，用户支付时，仅需输入CVV即可，从而避免像使用新卡支付时，输入卡号、有效期、CVV、持卡人姓名等完整卡要素信息。

### 推荐卡的优势
- 简化用户操作
- 提高支付成功率

### 集成模式支持情况

| 集成模式 | 是否支持推荐卡 |
|---------|--------------|
| 收银台模式 | 支持 |
| 纯API模式 | 不支持 |
| 前置组件模式 | 支持 |
| 链接支付模式 | 支持 |
