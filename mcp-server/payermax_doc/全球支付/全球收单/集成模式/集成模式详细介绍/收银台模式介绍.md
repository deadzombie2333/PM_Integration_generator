# 收银台模式介绍

PayerMax Hosted Payment Page是指PayerMax已构建（Pre-built）的H5支付页面，已经在其转化页面进行了精细的优化。商户无需开发收银台页面，完全由PayerMax提供，可根据设备进行页面自适应，极大简化商户集成，支持商户快速上线。

## 1. 适用性

- ✅ 适用APP或网页类商户，或者存在多端设备的商户
- ✅ 少量的对接成本，新上支付方式无需或极少改动即可支持
- ✅ 无需PCI-DSS compliant

## 2. 用户流程介绍

标准收银台分为以下两种类型，可根据您的收银台展示进行选择。

### 2.1 全量支付方式收银台

商户侧只需要提供checkout入口，用户点击后跳转进入PayerMax Hosted Payment Page进行支付方式的选择及支付确认。

**流程图：**
```
商户页面 → [Checkout按钮] → PayerMax收银台 → [用户选择支付方式] → 支付确认 → 完成支付
```

**特点：**
- 商户只需提供一个跳转入口
- 用户在PayerMax页面选择所有可用支付方式
- 支付方式由PayerMax管理和展示
- 适合快速集成场景

### 2.2 指定支付方式收银台

商户侧提供支付方式选择，用户可在商户收银台确认使用的支付方式后，进入支付流程。

**流程图：**
```
商户页面 → [用户选择支付方式] → 商户确认 → PayerMax收银台 → [指定支付方式] → 完成支付
```

**特点：**
- 商户自己展示支付方式列表
- 用户在商户页面选择支付方式
- 跳转到PayerMax时已指定支付方式
- 适合需要自定义支付方式展示的场景

## 3. 支付方式推荐

PayerMax预构建的标准收银台中包含支付方式推荐能力，会根据用户的历史支付行为或偏好进行支付方式推荐，缩短用户选择时间，提升支付效率和成功率。

**推荐机制：**
- 基于用户历史支付行为
- 分析用户支付偏好
- 智能排序支付方式
- 提升转化率和成功率

## 4. 集成模式特点

### 4.1 性能

#### 自适应多端设备
自适应包括PC、移动、平板等设备端的支付行为。

**支持设备：**
- 💻 PC桌面端
- 📱 移动手机端
- 📲 平板设备
- 🖥️ 各种屏幕尺寸

#### 采用多种优化手段提升站点性能
通过数据压缩、缓存优化、并行加载、自适应等算法技术，有效降低收银台加载时间。

**优化技术：**
- 数据压缩
- 缓存优化
- 并行加载
- 自适应算法
- 资源预加载

#### 本地快速通道直连
将技术基建与当地渠道能力相结合，全球范围提供高速无缝的支付服务。

**优势：**
- 本地化部署
- 就近接入
- 降低延迟
- 提升响应速度

#### 弱网环境加载优化
针对网络访问速度较慢的部分新兴市场地区，PayerMax提供fast 3G弱网环境优化方案。

**优化方案：**
- Fast 3G环境适配
- 渐进式加载
- 关键内容优先
- 降级方案支持

### 4.2 信息承载

#### 商户Logo显示
满足商户自定义收银台需求，用户进入收银台页面感知明确。

**功能：**
- 展示商户品牌Logo
- 增强用户信任感
- 保持品牌一致性

#### 重点突出金额信息
强调金额信息，用户下单支付更安心。

**展示内容：**
- 订单总金额
- 币种信息
- 清晰可见的价格展示

#### 支持支付优惠等标签
可承载更多订单信息，如VAT、手续费等。

**支持信息：**
- VAT税费
- 手续费
- 优惠折扣
- 其他费用明细

#### 支付引导
提供详细的用户引导，适配本地化语言，并支持点击"More info"并在弹层中查看所有订单信息。

**引导功能：**
- 本地化语言支持
- 详细订单信息展示
- More info弹层
- 完整订单明细

#### 收银台二维码及网址
提供收银台当前页二维码及网址；在当前支付环境无法完成支付时，允许用户复制地址或扫码使用其他浏览器打开并完成支付。

**跨设备支付：**
- 生成当前页面二维码
- 提供页面URL
- 支持复制链接
- 扫码跨设备支付
- 浏览器切换支持

### 4.3 页面交互

#### 卡支付页面显示层级
推荐卡与添加卡置于同一页面操作，便于用户切换与聚焦。

**交互优化：**
- 已保存卡片列表
- 添加新卡入口
- 同页面切换
- 减少跳转层级

#### 实时银行卡验证
支持用户进行实时银行卡验证，提前识别错误卡信息，防止交易失败。

**验证功能：**
- 实时卡号验证
- Luhn算法校验
- 卡类型识别
- 即时错误提示
- 减少支付失败

#### 卡账号列表显示
便捷直观的浏览历史使用账号并选用。

**列表功能：**
- 历史卡片展示
- 卡片尾号显示
- 卡类型图标
- 快速选择

#### CVV填写
用户可以便捷的在卡支付页面直接输入CVV，无需弹窗输入。

**优化体验：**
- 页面内直接输入
- 无需额外弹窗
- 流程更顺畅
- 减少操作步骤

#### 验证码收发交互
验证码收发逐步呈现，信息更加聚焦。

**交互流程：**
- 逐步展示验证步骤
- 信息聚焦展示
- 清晰的状态提示
- 倒计时显示

#### 收发问题处理
支持在验证码接收过程中修改手机号，便捷输错手机号场景处理；区分手机号错误及验证码错误，显示具体引导。

**错误处理：**
- 支持修改手机号
- 区分错误类型
- 具体错误提示
- 重新发送验证码
- 友好的错误引导

#### 支付引导页支付引导体验
根据不同支付流程，以场景化的交互步骤、指示图片及本地化语言等针对性地提供用户引导。

**引导内容：**
- 场景化交互步骤
- 可视化指示图片
- 本地化语言
- 分步骤引导
- 清晰的操作说明

### 4.4 页面视觉

页面简洁、聚焦，并支持深/浅色主题以及自定义UI风格（包括商户Logo、支付方式排序、主题色彩、边框圆角以及按钮风格等）。

#### 主题支持
- 🌞 浅色主题（Light Mode）
- 🌙 深色主题（Dark Mode）
- 自动适配系统主题

#### 自定义选项

**品牌定制：**
- 商户Logo展示
- 品牌色彩应用
- 自定义主题色

**布局定制：**
- 支付方式排序
- 页面元素布局
- 信息展示优先级

**样式定制：**
- 边框圆角设置
- 按钮风格选择
- 字体大小调整
- 间距控制

**视觉原则：**
- 简洁明了
- 信息聚焦
- 视觉层次清晰
- 符合本地审美

## 6. 适用场景

### ✅ 推荐使用场景
- 快速上线支付功能
- 多端设备支持需求
- 无前端开发资源
- 无PCI-DSS认证
- 需要专业优化的支付页面
- 频繁新增支付方式

### ⚠️ 不太适合场景
- 需要深度定制支付流程
- 需要页面内完成支付（不跳转）
- 有特殊的支付方式展示需求
- 需要完全控制用户数据
