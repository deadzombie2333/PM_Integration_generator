### Body (application/json)

| 参数名 | 类型 | 必填 | 说明 | 限制 |
|--------|------|------|------|------|
| code | string | 是 | 返回码,'APPLY_SUCCESS'代表成功 | <= 32 characters |
| msg | string | 是 | 响应描述,'Success.' | <= 256 characters |
| keyVersion | string | 是 | 密钥版本。当前值为:1 | <= 8 characters |
| appId | string | 否 | 商户app id | <= 64 characters |
| merchantNo | string | 是 | 商户Id | <= 15 characters |
| notifyTime | string | 是 | 请求时间,符合rfc3339规范,格式:yyyy-MM-dd'T'HH:mm:ss.SSSXXX | <= 32 characters |
| notifyType | string | 是 | 通知类型 CAPTURE | <= 16 characters |
| data | object | 是 | 通知数据体 | - |

#### data 对象

| 参数名 | 类型 | 必填 | 说明 | 限制 |
|--------|------|------|------|------|
| captureNo | string | 是 | Payermax生成的请款流水单号 | <= 64 characters |
| requestId | string | 是 | 商户请求id | <= 64 characters |
| outTradeNo | string | 是 | 商户外部订单号 | <= 32 characters |
| amount | string | 是 | 请款金额 | <= 1024 characters |
| currency | string | 是 | 请款币种 | - |
| status | string | 是 | 请款状态。SUCCESS、FAILED 、PENDING | - |
| isFinal | string | 是 | 是否最终请款 | - |
| createdTime | string | 是 | 创建时间 | - |
| succeededTime | string | 否 | 成功时间 | - |
| failedTime | string | 否 | 失败时间 | - |
| errorCode | string | 否 | 失败的错误码 | - |
| errorMsg | string | 否 | 失败的错误msg | - |

## Response

### 200 成功响应

#### Body (application/json)

| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| code | string | 是 | 返回码,'SUCCESS'代表成功 | SUCCESS |
| msg | string | 是 | 返回描述,'Success' | Success |
